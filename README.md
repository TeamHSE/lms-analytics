# LMS Analytics
## Краткое описание
**Платформа для школьников и студентов на data-driven
подходе**
- [x] Агрегирует информацию об успеваемости, поведению, SEL (социально-эмоциональное обучение), посещаемости
- [x] Предоставляет выводы и данные в дашборд панели.
- [ ] Интегрируема с любой внешней системой посредством предоставления публичного REST API
- [x] Акцент на простоте старта использования и целостности предоставления данных об обучающемся.

##Финансы
###Основные факторы стоимости привлечения клиента
1. Маркетинговые расходы: Затраты на рекламу, продвижение и другие маркетинговые активности. Это может включать контекстную рекламу, социальные сети, SEO и другие каналы.
2. Технологические расходы: Инвестиции в разработку и поддержку платформы, включая интеграцию с внешними системами через REST API. Использование API позволяет улучшить взаимодействие с клиентами и другими системами, что может снизить затраты на обслуживание12.
3. Аналитика и оптимизация: Платформы, использующие data-driven подход, требуют постоянного анализа данных о пользователях (успеваемость, поведение, посещаемость). Это может потребовать дополнительных ресурсов для сбора и обработки данных, что также влияет на стоимость привлечения клиента34.
4. Поддержка и обучение: Обеспечение поддержки пользователей и обучение их использованию платформы также добавляет к общим затратам. Упрощение старта использования платформы может помочь снизить эти затраты3.

##Проблемы
Платформа предназначена для решения следующих проблем:
1. Нет единой структуры данных в вузе
2. Нет согласовваности между участниками обр процесса
3. Нет полного понимания студентов
4. Студент выходит из ВУЗа потерянный, незаинтересованный
При этом она помогает:
1. Экономить время
2. Получать точные и актуальные данные об обр. процессах
3. Повышать качество образования через адаптацию между участниками процесса
4. Получать и использовать агрегированную информацию
